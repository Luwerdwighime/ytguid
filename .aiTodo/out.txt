= Общие настройки
Говори по-русски.

Девелоперская среда: исключительно Android, Termux, vim, root права; никаких ПК (считай, что челлендж - написать всё на мобиле).

Юзерская среда: обычный Андроид 9+, обычные права.

Отступы - 2 пробела, форматирование кода компактное (80 букв на строку - классика). Код пиши блоками, блоки doxy комментируй, старайся сушить код и дважды не делать одну логическую операцию в разных файлах/блоках. Длинные функции можно разбивать, если это не порождает функцию в одну строку. Использовать верблюжью нотацию, если нет иной культуры в данном вопросе. Полные пути в названиях файлов для комфортного копипаста. Файлы присылать целиком, а не кусками.

Сборка на ГитХабе, на рабочем телефоне только код, ключи, конфиги.

Название: ytguid.

Проект: https://github.com/Luwerdwighime/ytguid

Не забывай читать файлы из репы, если твой новый файл содержит меньше функционала, чем исходный, то я буду не доволен. Хотя, переписать ты можешь, я почитаю, но не деградируй решение. Всегда давай мне файлы целиком, куски кода мы можем обсуждать, но копипаст на мобиле хорош только целиком (6" экрана всего).

= Демон ytguid
Согласно новому ТЗ, gui андроид приложения разбивается на броузер и этот демон. Демон будет запущен самим java-приложением и будет слушать порт 27523, обслуживать юзера он будет в HTML по http.

Демона пишем на c++23, используем Crow, собираем через cmake. Исходники делим на cpp и hpp. vendors/Crow, vendors/asio и vendors/boost содержат полную клонированную репу каждого.

== Алгоритм работы
Наше дерево сайта:
/hello - Начальная страница "Hello world".
/run/[1-4] - один из 4х режимов работы, пока отвечаем "Режим #номер".
/opt/[1-4] - опции к одному из режимов, пока отвечаем "Опции #номер".

== Папка www
Папка www содержит HTML код шаблонов соответствующих страниц (hello.html, run[1-4].html, opt[1-4].html). Подобно PHP следует инжектить наши строки внутрь шаблона и возвращать юзеру через HTTP, готовую страницу.

== Опции
Через getopt() анализируем **argv. Опция -w, указывает относительный путь от исполняемого бинарника до папки www. Если - w не указана, то ищем папку www рядом с экзешником.
